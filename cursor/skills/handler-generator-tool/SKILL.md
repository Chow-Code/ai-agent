# Handler代码生成工具Skill

## 工具概述

Handler代码生成工具基于`.lp`接口文件（使用proto3语法）自动生成Handler代码。工具使用职责链模式，分为三个步骤：验证语法和消息类型、生成接口定义和注册代码、生成客户端辅助函数。

## 使用场景

- 新增或修改Handler接口定义（`.lp`文件）后需要重新生成代码
- 需要生成Handler接口定义时
- 需要生成客户端辅助函数时
- 修改服务间通信接口后需要更新代码

## 前置条件

- Go环境已配置
- 项目依赖已安装
- `.lp`文件已准备好（位于`proto/`目录）
- `.pb.go`文件已生成（位于`message/`目录）

## 命令格式

```bash
cd tools/handler-generator
go run main.go -proto ../../proto -message ../../message -output ../..
```

### 参数说明

- `-proto`: proto文件目录（包含.lp文件），默认: `proto`
- `-message`: message目录（已生成的.pb.go文件），默认: `message`
- `-output`: 输出目录（必需），生成的代码将输出到`{output}/common/handlergen/`目录

## 工具功能

### 第一步：验证语法和消息类型

- 验证`.lp`文件语法
- 检查消息类型是否存在（包括定义）

### 第二步：生成接口定义和注册代码

- 生成Handler接口定义
- 生成Register方法和handleXXX方法
- 用于服务自己写代码

### 第三步：生成客户端辅助函数

- 生成SendXXX方法
- 自动处理消息序列化、BaseMsg创建、waypoint转发等
- 用来发消息

## 职责链流程

```
ValidateHandler（验证）
  ├─ 扫描.lp文件
  ├─ 解析.lp文件语法
  └─ 验证消息类型是否存在
      ↓
InterfaceGeneratorHandler（生成接口）
  ├─ 生成Handler接口定义
  └─ 生成Register方法和handleXXX方法
      ↓
ClientGeneratorHandler（生成客户端）
  └─ 生成SendXXX辅助函数
```

## 常见问题

### Q: .lp文件格式是什么？

A: `.lp`文件使用proto3语法，定义Handler接口。示例：

```protobuf
// proto/game/login_handler.lp
service LoginHandler {
  // HandleLogin 处理登录请求
  // 消息名称: "MsgCtrReqLogin", "CtrSvr.MsgCtrReqLogin"
  rpc HandleLogin (MsgCtrReqLogin) returns (MsgCtrResLogin);
}
```

### Q: 生成的代码会被覆盖吗？

A: 是的，代码生成工具会**直接覆盖**已存在的生成文件，这是标准行为。生成的文件都包含`"Code generated by handler-generator. DO NOT EDIT."`注释。

### Q: 如何更新Handler实现？

A: 根据生成的接口更新Handler实现，确保实现所有接口方法。Handler必须实现`common/handlergen`中生成的接口。

## 最佳实践

1. **先运行Proto工具**：在运行Handler生成工具之前，先运行Proto代码生成工具
2. **检查生成结果**：生成后检查是否有编译错误
3. **实现接口方法**：根据生成的接口实现Handler方法
4. **使用生成的客户端函数**：发送消息到其他服务时，必须使用生成的客户端函数
