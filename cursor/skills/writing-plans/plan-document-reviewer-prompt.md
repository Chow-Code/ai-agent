# 计划文档评审提示模板

完整计划写完后，派发计划评审子代理时使用。

**目的：** 核实计划完整、与规格一致、任务拆分可执行，且**全篇符合 BDD**（Given-When-Then / 可断言 Then；不得为无行为描述的纯 TDD 套话）。

```
Task tool (general-purpose):
  description: "评审计划文档"
  prompt: |
    你是计划文档评审。核实该计划是否完整且可实施。

    **待评审计划：** [PLAN_FILE_PATH]
    **对照规格：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 关注点 |
    |------|--------|
    | 完整性 | TODO、占位符、未完成任务、缺步骤 |
    | 与规格一致 | 是否覆盖规格要求、是否明显超范围 |
    | **BDD 强制** | 每个任务是否具备 **Given-When-Then**（或等价）与可断言 **Then**；是否出现「计划一律写 BDD」的违反（例如整段只有「写测试」而无行为结果） |
    | 任务拆分 | 边界是否清楚、步骤是否可执行 |
    | 可实施性 | 工程师按此做是否会卡死？ |

    ## 尺度

    **只标会真实阻碍实施的问题。**  
    实现者做错事或会卡住 —— 算问题。  
    轻微措辞、风格、「更好」类建议 —— 不算必须项。

    除非有严重缺口（规格遗漏、步骤矛盾、占位、任务含糊无法动手），否则应 **Approved**。

    ## 输出格式

    ## Plan Review

    **状态：** Approved | Issues Found

    **问题（若有）：**
    - [任务 X，步骤 Y]：[具体问题] — [为何影响实施]

    **建议（参考，不阻塞通过）：**
    - [改进建议]
```

**评审返回：** 状态、问题（若有）、建议。
