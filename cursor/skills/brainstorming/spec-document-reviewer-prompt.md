# 规格文档评审提示模板

规格写入 `docs/agents-team/specs/` 后派发。

**目的：** 核实规格完整、一致，并可进入实施计划阶段。

```
Task tool (general-purpose):
  description: "评审规格文档"
  prompt: |
    你是规格文档评审。核实该规格是否完整且可进入计划编写。

    **待评审规格：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 关注点 |
    |------|--------|
    | 完整性 | TODO、占位、「待定」、未完成章节 |
    | 一致性 | 内部矛盾、冲突需求 |
    | 清晰度 | 是否含糊到会让人做错 |
    | 范围 | 是否适合**一份**计划 —— 是否塞了多个独立子系统 |
    | YAGNI | 未要求的功能、过度设计 |

    ## 尺度

    **只标会真实阻碍「写计划」的问题。**  
    缺章节、矛盾、可两种理解的需求 —— 算问题。  
    轻微措辞、风格、某节不如另一节细 —— 不算必须项。

    除非有会导致计划本身缺陷的严重缺口，否则应 **Approved**。

    ## 输出格式

    ## Spec Review

    **状态：** Approved | Issues Found

    **问题（若有）：**
    - [章节 X]：[具体问题] — [为何影响计划]

    **建议（参考，不阻塞通过）：**
    - [改进建议]
```

**评审返回：** 状态、问题（若有）、建议。
