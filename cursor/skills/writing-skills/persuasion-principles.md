# 技能设计中的说服原则（概要）

## 概述

大模型与人类类似，会受常见说服因素影响。理解这些有助于设计更有效的技能——目的不是操纵，而是让关键实践在压力下仍被遵守。

**研究参考：** Meincke 等 (2025) 在大量对话中验证多种说服原则对合规率的影响（可配合原英文文献阅读）。

## 七项原则（与技能结合）

### 1. 权威（Authority）
- **技能中：** 祈使句、「必须/绝不」、无例外表述，减少纠结与合理化。  
- **适用：** 纪律类技能（BDD 场景/验证、完成前验证）、安全相关、公认最佳实践。

### 2. 承诺一致（Commitment）
- **技能中：** 要求宣告「正使用某技能」、显式选项、Todo 勾选。  
- **适用：** 多步流程、需要可追溯时。

### 3. 稀缺（Scarcity）
- **技能中：** 「继续前必须…」「完成后立即…」等时限与顺序。  
- **适用：** 立即验证、防「以后再补」。

### 4. 社会认同（Social Proof）
- **技能中：** 「每次」「总是」、常见失败模式表。  
- **适用：** 统一规范、警示常见坑。

### 5. 好感 / 认同（Liking / Unity）
- **技能中：** 适度即可；纪律技能仍以清晰规则为主。

### 6. 互惠（Reciprocity）
- **技能中：** 较少直接使用；可强调「遵守流程 = 减少返工」。

### 7. 其他变体
- 以原英文版与论文为准；本文件为工作区使用的**中文摘要**。
