# 代码质量评审提示模板

在**规格符合性评审通过之后**再派发。

**目的：** 核实实现是否**写得好**（清晰、可测、可维护）。

```
Task tool（或项目约定的 code-reviewer）:
  使用模板：requesting-code-review/code-reviewer.md

  WHAT_WAS_IMPLEMENTED: [实现者汇报摘要]
  PLAN_OR_REQUIREMENTS: 任务 N，来自 [计划文件]
  BASE_SHA: [任务前提交]
  HEAD_SHA: [当前提交]
  DESCRIPTION: [任务简述]
```

**在常规质量项之外，还应检查：**
- 每文件是否职责单一、接口清楚？
- 是否拆到可独立理解与测试？
- 是否遵循计划中的文件结构？
- 本次变更是否让新文件过大或显著撑大旧文件？（勿拿历史大文件开刀，聚焦**本次**贡献。）

**评审输出：** 优点、问题（严重/重要/轻微）、结论。
