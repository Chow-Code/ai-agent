# 规格符合性评审提示模板

派发「规格符合性」子代理时使用。

**目的：** 核实实现者做的是**被要求**的（不多不少）。

```
Task tool (general-purpose):
  description: "评审任务 N 的规格符合性"
  prompt: |
    你在核实实现是否符合规格。

    ## 要求是什么（来自 BDD 实施计划中的本任务：Given-When-Then / Then）

    [任务全文：须含可验收 Then]

    ## 实现者声称做了什么

    [来自实现者汇报]

    ## 关键：不要相信汇报 alone

    实现若完成得「可疑地快」，汇报可能不全、不准或过于乐观。你必须**独立**核实。

    **不要：**
    - 轻信其实现内容
    - 轻信其完整性说法
    - 接受其对需求的单方面解读

    **要：**
    - 阅读其**实际写的代码**
    - 逐条对照需求
    - 查是否声称做了但未实现
    - 查是否有多余未提及功能

    ## 职责

    读代码并核实：

    **遗漏：**
    - 是否实现全部要求？
    - 有无跳过？
    - 是否声称可用实际未做？

    **多余：**
    - 是否做了未要求的？
    - 是否过度设计或加「锦上添花」？

    **误解：**
    - 是否理解偏了？是否题不对？

    **以读代码为准，不以汇报为准。**

    汇报格式：
    - ✅ 规格符合（经代码检查后）
    - ❌ 问题：[具体缺什么/多什么，附 文件:行]
```
