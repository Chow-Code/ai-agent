## 评审清单

**代码质量：**  
职责分离？错误处理？类型安全（如适用）？DRY？边界情况？

**架构：**  
设计是否合理？扩展与性能？安全隐患？

**测试：**  
是否在测真实逻辑（而非只测 mock）？边界与集成？全绿？

**需求：**  
计划项是否都满足？与规格一致？无范围蔓延？破坏性变更有说明？

**生产就绪：**  
有无迁移策略？兼容性？文档？明显 bug？

## 规则要点

**要做：** 按真实严重程度分类；具体（文件:行）；说明为何；肯定优点；给明确结论。  

**不要做：** 没细看就说「不错」；把琐事标成严重；没读代码就给意见；泛泛的「加强错误处理」；不给明确结论。
