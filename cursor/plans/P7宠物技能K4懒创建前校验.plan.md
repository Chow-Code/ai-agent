# P-7 宠物技能：懒创建前校验（主修复点在 pet_skill_level_cost.go）

## 根因（用户指出）

`[pet_skill_level_cost.go](d:\Project\TL3Server\srv\game\domain\pet\service\pet_skill_level_cost.go)` 在 `**ComputePetSkillLevelUpCost**`（约 91–95 行）与 `**ApplyPetSkillLevelUp**`（约 152–156 行）中，在确认 `skillId` 是否为合法宠物升级技能之前，就对 `Skills[skillId]` 做 **Level=0 占位写入**。客户端传入错误 id（如 `CtPetSkill` 配置 Id `10001`、`20003001` 等）仍会落库，登录后 `PlayerInfo.Skills` 出现脏数据。

## 合法性（与 P-7 一致）

- `**skillId` 合法** 当且仅当 `CtPetSkillUp` 中**至少存在一行** `row.SkillId == skillId`（与现有 `TryGetValue2(skillId, L)` 所用配置域一致）。
- **不做**以 `CtPetSkill.Id` 为准的兼容（避免再次接受 `10001` 类错误主键）。

## 实现要点

1. **小函数**（同文件或 `pet_skill_module_sanitize.go` 二选一，优先同文件减少跳转）
  - `func petSkillIdDefinedInCtPetSkillUp(skillId int32) bool`：遍历 `excel.CfgManager.CtPetSkillUp.GetList()` 判 `SkillId`；可用 `sync.Once` 建 `map[int32]struct{}` 避免热路径反复扫表（表规模小也可先直接扫，后续再优化）。
2. `**ComputePetSkillLevelUpCost`**（`count > 0` 才会走到懒创建）
  - 在 **写入** `Skills[skillId]` 之前：若 `sk == nil` 且 `!petSkillIdDefinedInCtPetSkillUp(skillId)`，则 `**return nil, PetSkillLevelMax`**（与「无配置行」语义一致，或若团队更希望「非法参数」可改为 `PetNotFound`，二选一则全文统一）。  
  - **禁止**在非法 id 下创建 map 条目。
3. `**ApplyPetSkillLevelUp`**
  - 同样在懒创建前做同上校验；非法则返回对应错误且 **不写 map**。  
  - `**count == 0` 且 `sk == nil`**：当前代码会先占位再返回，会污染 K-4；应改为 **不创建**、直接 `return None, nil, 0`（或仅返回已有 `sk`，无则 nil）。
4. **范围外**
  - **不**做「未上线兼容」的 Load 后全量净化与 `PetServiceFactory` 强制 Save（除非你后续明确要求清 Redis 存量）。
5. **测试与文档**
  - 单测：非法 `skillId` 调用 `ComputePetSkillLevelUpCost`（`count>0`）后 `**SkillModule.Skills` 不含该 key**；合法路径不变。  
  - `[srv/game/domain/pet/README.md](d:\Project\TL3Server\srv\game\domain\pet\README.md)` 增加一句：P-7 仅允许 `CtPetSkillUp` 已声明的 `SkillId` 进入 K-4。
